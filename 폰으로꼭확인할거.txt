await 이거 정리는 했는데, 예제 파악하기
비동기 함수를 일시 중지하고 다시 시작하며, 프로미스의 완료/거부를 기다립니다.

자바스크립트에서 await는 Promise가 처리(성공 또는 실패)될 때까지 기다리게 하는 키워드입니다.
이를 통해 비동기 작업을 마치 동기 작업처럼 순차적이고 직관적으로 작성할 수 있습니다.
주요 특징과 동작 방식은 다음과 같습니다.
1. 주요 기능
실행 일시 중지 및 재개: await를 만나면 해당 비동기 작업이 끝날 때까지 기다린 후(대기), 
완료되면 결과값을 반환하고 다음 코드를 실행합니다.
스레드 차단 안 함: 기다리는 동안 자바스크립트 메인 스레드는 멈추지 않고 다른 작업을 계속 수행하므로(논블로킹), 
프로그램이 멈추지 않습니다.
2. 필수 조건
await는 반드시 async 키워드가 붙은 함수 내부에서만 사용할 수 있습니다.
보통 서버에서 데이터를 받아오거나(API 호출), 파일 읽기 등 시간이 걸리는 작업 앞에 사용합니다.
3. 간단한 예제 코드javascript
async function fetchUserData() {
  // 1. 서버에 요청을 보내고 응답이 올 때까지 대기
  const response = await fetch('https://example.com');
  
  // 2. 응답 데이터를 JSON 형태로 변환할 때까지 대기
  const user = await response.json();
  
  // 3. 변환된 데이터를 바탕으로 출력
  console.log(user.name);
}
4. 에러 처리await로 기다리던 Promise가 실패(거부)할 경우 에러가 발생하므로,
이를 처리하려면 try...catch 문을 함께 사용해야 안전합니다.javascript
async function getData() {
  try {
    const data = await fetchApi();
    console.log(data);
  } catch (error) {
    console.error('에러 발생:', error);
  }
}

in --> 이거 정리는 했는데, 예제 파악하기
연산자 in는 객체가 주어진 속성을 가지고 있는지 여부를 판단합니다.

자바스크립트에서 in은 객체 안에 특정 속성(키)이 존재하는지 확인하는 연산자와 객체의 속성들을 순회하는 반복문 두 가지로 사용됩니다.
1. in 연산자 (속성 존재 확인)객체 안에 해당 키(key)가 있는지 확인하여 true 또는 false를 반환합니다.
구문: 속성_이름 in 객체_이름javascript
const user = { name: "홍길동", age: 20 };

console.log("name" in user); // true
console.log("address" in user); // false
코드를 사용할 때는 주의가 필요합니다.더 자세한 내용은 MDN in 연산자 문서에서 확인할 수 있습니다.
2. for...in 반복문 (객체 속성 순회)객체가 가지고 있는 모든 열거 가능한 속성들을 순회할 때 사용합니다. 
주로 객체의 키(key)를 가져올 때 씁니다.
구문: for (let key in 객체_이름) { ... }javascript

const user = { name: "홍길동", age: 20 };

for (let key in user) {
  console.log(key); // name, age 출력
  console.log(user[key]); // 홍길동, 20 출력
}

?? --> 이거 정리는 했는데, 예제 파악하기
널리시 병합 연산자.

자바스크립트에서 ??는 Nullish 병합 연산자(Nullish Coalescing Operator)를 의미합니다.
좌측 피연산자 ?? 우측 피연산자 형태로 사용되며,
좌측 값이 null 또는 undefined일 때만 우측 값을 반환하고, 그렇지 않으면 좌측 값을 반환합니다.
주요 특징 및 예시
기본 사용법: 값이 확정되지 않은 변수에 기본값(default)을 안전하게 할당할 때 유용합니다.
javascript
const name = null;
const defaultName = "Guest";
console.log(name ?? defaultName); // "Guest" 출력
코드를 사용할 때는 주의가 필요합니다.
|| (OR) 연산자와의 차이점: || 연산자는 0, ""(빈 문자열), false 등도 모두 '거짓(Falsy)'으로 판단하여 무시하지만, 
??는 오직 null과 undefined만 건너뜁니다.javascriptconst count = 0;

// OR 연산자 사용
console.log(count || 10); // 10 출력 (0을 falsy로 인식)

// Nullish 병합 연산자 사용
console.log(count ?? 10); // 0 출력

[a, b] = arr,{ a, b } = obj
구조 분해 할당을 사용하면 배열이나 객체의 속성을 배열이나 객체 리터럴과 유사한 구문을 사용하여 변수에 할당할 수 있습니다.

구조분해 할당(Destructuring Assignment)은 배열이나 객체의 속성을 해체하여 그 값을 개별 변수에 맞춤형으로 담을 수 있게 하는 자바스크립트 표현식입니다. [1, 2] 
코드를 더 간결하게 만들고 가독성을 높여줍니다. [3, 4, 5]

private --> 이거 정리는 했는데, 예제 파악하기

자바스크립트에서 private은 클래스 내부의 변수나 메서드를 외부에서 접근하지 못하도록 숨기는 기능(캡슐화)을 의미합니다. 
변수나 메서드 이름 앞에 # 기호를 붙여 공식적으로 private 속성을 지원합니다.
1. 공식 문법: # (Private 클래스 필드)이름 앞에 #을 붙이면 클래스 내부에서만 사용할 수 있으며, 
외부 인스턴스에서는 읽거나 수정할 수 없습니다.javascript
class Counter {
  #count = 0; // private 변수 선언

  increment() {
    this.#count++; // 내부에서는 접근 가능
  }

  getCount() {
    return this.#count;
  }
}

const myCounter = new Counter();
myCounter.increment();
console.log(myCounter.getCount()); // 1
// console.log(myCounter.#count); // 에러 발생! 외부에서 직접 접근 불가

클로저(Closure)를 활용한 방식함수 스코프의 특징을 활용해 외부에 노출되지 않는 private 변수를 생성하는 방식입니다.javascript  이건 정리 안함 모름,...
function createCounter() {
  let count = 0; // 외부에서 접근 불가능한 변수
  return {
    increment: function() { count++; },
    getCount: function() { return count; }
  };
}

const counter = createCounter();

public --> 이거 정리는 했는데, 예제 파악하기

public은 클래스나 객체 내부의 변수(프로퍼티)와 함수(메서드)를 외부에서 자유롭게 접근하고 수정할 수 있도록 공개하는 상태를 의미합니다.
기본적으로 자바스크립트는 객체나 클래스 내부의 모든 멤버를 기본적으로 public으로 취급합니다.
기본 개념 (ES6 클래스 기준)자바스크립트 클래스에서 별도의 키워드 없이 변수나 메서드를 선언하면 기본적으로 모두 외부에서 접근 가능한 public이 됩니다.
javascript
class User {
  constructor(name) {
    this.name = name; // public 프로퍼티 (외부 접근 가능)
  }

  sayHello() { // public 메서드 (외부 접근 가능)
    console.log(`안녕하세요, ${this.name}입니다.`);
  }
}

const user = new User("홍길동");
console.log(user.name); // 외부에서 직접 접근 가능: "홍길동"
user.sayHello();        // 외부에서 직접 호출 가능

 명시적인 Public 필드 문법 (#과의 차이)최신 자바스크립트(ES2022 이상)에서는 private(비공개)를 나타낼 때 변수명 앞에 #을 붙이도록 표준화되었습니다. 
#이 붙지 않은 일반 변수들은 명시적인 public 상태입니다.javascript
class Animal {
  name = "강아지";       // public 필드 (외부 접근 가능)
  #age = 3;            // private 필드 (클래스 내부에서만 접근 가능)

  getAge() {
    return this.#age;  // 클래스 내부에서는 private 접근 가능
  }
}

const dog = new Animal();
console.log(dog.name); // "강아지" (정상 작동)
console.log(dog.#age); // 에러 발생! 외부에서 private 접근 불가

throw --> 이거 정리는 했는데, 예제 파악하기

throw는 프로그래머가 고의로 에러(예외)를 발생시키는 키워드입니다.
문법적으로 throw 에러_객체; 형태로 사용하며, 이 코드가 실행되면 즉시 프로그램의 실행이 중단되고 호출 스택의 상위로 에러를 던집니다. 
throw의 핵심 역할의도적인 에러 발생: 잘못된 입력값 등 프로그램이 정상적으로 진행될 수 없는 상황에서 에러 객체를 생성하여 시스템에 알립니다.
코드 실행 즉시 중단: throw문 아래에 있는 코드는 더 이상 실행되지 않고 제어 흐름이 즉시 catch문으로 넘어갑니다.
자주 사용하는 형태1. 에러 객체(Error) 던지기가장 표준적이고 권장되는 방식입니다. 
에러 이름과 상세 메시지를 담을 수 있습니다.javascript
function divide(a, b) {
  if (b === 0) {
    // 0으로 나눌 수 없으므로 고의로 에러 발생시키기
    throw new Error("0으로 나눌 수 없습니다."); 
  }
  return a / b;
}

2. 던진 에러 처리(try...catch)throw로 던져진 에러는 try...catch문을 통해 잡아내어 프로그램이 죽지 않도록 안전하게 후속 처리할 수 있습니다.
javascript
try {
  divide(10, 0); // 에러 발생 지점
} catch (error) {
  console.error("에러 발생:", error.message); 
} finally {
  console.log("작업이 완료되었습니다."); // 에러 유무와 관계없이 실행
}

3. 숫자, 문자열, 커스텀 객체 던지기Error 객체 외에도 원시값이나 임의의 객체를 던질 수도 있습니다.
javascript// 문자열 던지기
throw "잘못된 접근입니다.";

// 커스텀 객체 던지기
throw { errorCode: 404, message: "페이지를 찾을 수 없습니다." };

static --> 이거 정리는 했는데, 예제 파악하기


static은 객체가 아닌 클래스 자체에 속하는 메서드나 속성을 정의할 때 사용합니다. 
인스턴스를 생성하지 않고도 클래스 이름으로 직접 호출할 수 있어, 
주로 유틸리티 함수에 활용됩니다.
자바스크립트에서 static의 주요 특징과 사용 방법은 다음과 같습니다.
1. 정적 메서드 (Static Method)static 키워드를 앞에 붙여 선언하며, 
클래스의 인스턴스(객체)를 만들지 않고 클래스명.메서드명() 형태로 호출합니다.
호출 불가: 인스턴스에서 정적 메서드를 호출하려고 하면 에러가 발생합니다.
this 키워드: 정적 메서드 내부의 this는 인스턴스가 아닌 클래스 자체를 가리킵니다.javascript
class MathUtil {
  static sum(a, b) {
    return a + b;
  }
}

// 인스턴스 생성 없이 바로 호출
console.log(MathUtil.sum(5, 10)); // 15
코드를 사용할 때는 주의가 필요합니다.2. 정적 속성 (Static Property)클래스 레벨에서 공통으로 사용되는 데이터를 저장할 때 사용합니다.
javascript
class Config {
  static API_URL = "https://example.com";
}

console.log(Config.API_URL); // "https://example.com"

async function --> 이거 정리는 했는데, 예제 파악하기
이는 async function비동기 함수 표현식을 정의합니다.

async 키워드는 함수가 비동기적으로 작동하며 항상 Promise 객체를 반환하도록 선언합니다. 
함수 내부에서 await 키워드를 사용해 비동기 작업이 완료될 때까지 기다리게 할 수 있어, 
복잡한 코드 없이도 동기식 코드처럼 읽기 쉽게 비동기 로직을 작성할 수 있습니다.
주요 특징
항상 Promise 반환: async 함수는 일반 값을 반환해도 암묵적으로 Promise.resolve()로 감싸서 반환합니다.
await 사용 가능: async 함수 내부에서만 await 키워드를 사용할 수 있습니다.
await는 Promise가 처리될 때까지(이행되거나 거부될 때까지) 코드 실행을 일시 중지합니다.
예외 처리: 동기식 코드에서 사용하는 try...catch 문을 그대로 사용하여 비동기 에러를 처리할 수 있습니다.
기본 사용법javascript// 
1. 일반적인 async function 선언
async function fetchData() {
    try {
        // fetch()가 완료될 때까지 기다렸다가 response 변수에 할당
        let response = await fetch('https://example.com');
        
        // JSON 파싱을 기다림
        let data = await response.json();
        return data;
    } catch (error) {
        console.error('데이터를 가져오는 데 실패했습니다:', error);
    }
}

// 2. 화살표 함수에 적용
const fetchUsers = async () => {
    const response = await fetch('https://example.com');
    return response.json();
};

var users = { tj: { name: 'tj' } };는 Node.js(JavaScript)에서 간단한 인메모리(In-Memory) 사용자 데이터베이스 역할을 하는 객체를 선언한 것입니다.
데이터베이스(MySQL, MongoDB 등)를 아직 연동하지 않은 테스트 단계나 학습용 코드에서 회원 정보를 임시로 저장하고 조회하기 위해 자주 사용하는 패턴입니다.
## 🔑 구조 및 문법적 의미

* 중첩 객체(Nested Object) 구조: users라는 대형 객체 안에, 사용자 아이디인 tj를 Key(키)로 사용하고, 그 안에 해당 사용자의 상세 정보 객체({ name: 'tj' })를 Value(값)로 매핑한 구조입니다.
* 인메모리 데이터: 서버의 메모리(RAM)에 직접 저장되므로 속도가 매우 빠릅니다. 하지만 서버를 껐다 켜면 데이터가 모두 초기화(증발)된다는 특징이 있습니다.

## 🔄 실제 활용 시나리오 (예시 플로우)
앞서 질문하신 세션, 로그인 미들웨어 흐름과 연결하면 주로 사용자 조회 및 인증에 사용됩니다.
1. 로그인 확인 (조회)
사용자가 로그인 창에 아이디로 tj를 입력하면, 서버는 이 객체를 조회하여 가입된 회원인지 확인합니다.

var username = 'tj'; // 사용자가 입력한 아이디
if (users[username]) {
    console.log('회원 정보 발견:', users[username].name); // 'tj' 출력
} else {
    console.log('존재하지 않는 회원입니다.');
}

2. 회원가입 (추가)
새로운 회원(예: 아이디 gildong)이 가입하면 users 객체에 동적으로 새로운 프로퍼티를 추가합니다.

users['gildong'] = { name: '홍길동' };
// 결과 users 객체:// {//   tj: { name: 'tj' },//   gildong: { name: '홍길동' }// }

const obj = { key: "value" }; 코드는 자바스크립트에서 상수 객체를 선언하고 초기화하는 문장입니다.
## 📌 코드의 상세 구성 의미

* const: 재할당이 불가능한 상수를 선언하는 키워드입니다. 한 번 할당하면 obj = ... 형태로 다른 값을 다시 대입할 수 없습니다. [1, 2] 
* obj: 생성된 객체를 가리키는 변수(식별자)의 이름입니다. [3] 
* = : 대입 연산자입니다. 우변의 객체를 좌변의 변수에 할당합니다.
* { key: "value" }: 자바스크립트의 객체 리터럴 표현식입니다. 중괄호 {}를 사용해 객체를 직접 생성했습니다.
* key: 객체의 속성 이름(Property Key)입니다.
   * "value": 해당 속성에 저장된 문자열 데이터(Property Value)입니다. [4, 5, 6, 7, 8]

## ⚠️ 핵심 주의점 (중요)
const로 선언했기 때문에 객체 자체를 바꾸는 것은 불가능하지만, 객체 내부의 속성을 변경하는 것은 가능합니다. [9, 10] 

* 불가능한 작업 (오류 발생):

obj = { name: "kim" }; // TypeError: Assignment to constant variable.

* 가능한 작업:

obj.key = "new value"; // 내부 값 변경 가능
obj.newKey = "hello";   // 새로운 속성 추가 가능

자바스크립트의 with 문은 특정 객체를 스코프 체인에 추가하여, 객체의 속성이나 메서드를 호출할 때 객체 이름을 생략할 수 있게 해주는 문법입니다.
하지만 혼란스러운 버그를 유발하고 성능 최적화를 방해하며 엄격 모드(strict mode)에서는 사용이 금지되어 있어 현재는 사용이 권장되지 않습니다.
사용 예시 (권장하지 않음)
javascript
const circle = { radius: 10, color: 'red' };

// with를 사용하면 객체 이름을 생략할 수 있습니다.
with (circle) {
  console.log(radius); // 10
  console.log(color);  // red
}
.권장되는 대체 방법with 문을 사용하는 대신 객체 구조 분해 할당(Destructuring assignment) 을 사용하면 코드를 훨씬 안전하고 깔끔하게 작성할 수 있습니다.
javascript
const circle = { radius: 10, color: 'red' };

// 구조 분해 할당을 사용하여 속성을 가져옵니다.
const { radius, color } = circle;

console.log(radius); // 10
console.log(color);  // red

const delay = (ms) => new Promise(resolve => setTimeout(resolve, ms));

// 비동기 제너레이터 함수 선언
async function* asyncGenerator() {
  console.log("시작!");
  
  for (let i = 1; i <= 3; i++) {
    await delay(1000); // 1초 대기
    yield i; // 값 반환
  }
  
  console.log("종료!");
}

// 순회하기
async function main() {
  for await (const value of asyncGenerator()) {
    console.log(value); // 1초 간격으로 1, 2, 3 출력
  }
}

main();
