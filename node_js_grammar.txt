const app = express(); 코드는 Node.js에서 웹 서버를 구축하기 위해 Express 프레임워크의 인스턴스(애플리케이션 객체)를 생성하고 초기화하는 역할을 합니다.이를 통해 서버의 뼈대를 잡고 다양한 기능과 설정을 다룰 수 있게 됩니다.핵심 기능 및 역할서버 설정: 포트 번호(app.set), 뷰 엔진 등 애플리케이션의 전반적인 설정을 정의합니다.라우팅: 사용자가 접속하는 경로(URL)와 HTTP 메서드(GET, POST 등)에 따라 서버가 어떻게 응답할지 결정합니다.미들웨어 연결: 서버 요청(Request)과 응답(Response) 사이클에서 필요한 기능(쿠키 파싱, 본문 데이터 파싱 등)을 app.use()로 추가할습니다.서버 구동: 정의된 설정을 바탕으로 실제 네트워크 요청을 수신 대기 상태로 만듭니다.

get() ===> 예시: cosnt app = express(); app.get()(() => {}) 클라이언트(웹 브라우저)의 GET 요청을 처리하는 라우팅 메서드입니다. 주로 서버로부터 HTML 파일, 데이터 등 리소스를 조회하거나 가져올 때 사용됩니다. URL 주소와 일치하는 요청이 들어오면 정의된 콜백 함수가 실행됩니다.1. 주요 역할라우팅: 특정 URL(예: /, /about)로 들어오는 요청을 가로채어 원하는 로직으로 연결합니다.데이터 조회: 서버의 데이터를 읽어와 사용자에게 보여주는 역할을 담당합니다.데이터 전달: URL 뒤에 쿼리 스트링(?name=nodejs)을 붙여 서버에 파라미터를 전달할 때 사용됩니다.

req와 res ===> 서버가 클라이언트와 통신할 때 사용하는 핵심 객체입니다.req (Request): 클라이언트가 서버로 보낸 요청 정보를 담습니다.res (Response): 서버가 클라이언트에게 보낼 응답 정보를 담습니다.

res.send("hello groom"); ===> 이 코드는 Node.js의 Express 프레임워크에서 클라이언트(브라우저 등)의 HTTP 요청에 대해 "hello groom"이라는 텍스트 데이터를 응답으로 보내는 명령입니다.

res.status(200); ===> Node.js(주로 Express 프레임워크)에서 res.status(200)는 HTTP 상태 코드(Status Code)를 200 (OK) 으로 명시적으로 설정하는 역할을 합니다.

setTimeout ===> 일정 시간이 지난 후 특정 코드나 함수를 단 한 번 실행하도록 예약하는 타이머 함수입니다
setTimeout(실행할 함수, 지연시간_ms, 인수1, 인수2...);
