require('pbkdf2-password')() -->


Node.js에서 require('pbkdf2-password')()는 pbkdf2-password 모듈을 불러온 뒤, 곧바로 함수로 실행하여 암호화/검증 옵션이 적용된 인스턴스를 생성하는 코드입니다. [1, 2] 
이 코드는 주로 사용자의 비밀번호를 안전하게 해싱(암호화)하고, 로그인 시 비밀번호가 일치하는지 검증할 때 사용합니다. [3] 
## 🛠️ 코드 구조 뜯어보기

* require('pbkdf2-password'): 외부 라이브러리인 pbkdf2-password 모듈을 가져옵니다. 이 모듈은 내부적으로 crypto를 사용하여 편리하게 비밀번호를 처리해 줍니다.
* (): 가져온 모듈이 '함수' 형태이기 때문에, 뒤에 괄호를 붙여 즉시 실행합니다. 실행 시 기본 설정(옵션)이 적용된 객체가 반환됩니다. [4] 

보통 아래처럼 변수에 할당하여 사용합니다: [1] 

const hasher = require('pbkdf2-password')();

------------------------------
## 💡 주요 기능 및 특징

* Salt 자동 생성: 레인보우 테이블 공격(미리 계산된 해시 테이블)을 막기 위해, 매번 고유한 솔트(Salt) 값을 자동으로 생성하여 안전하게 결합합니다. [5] 
* 반복 해싱(Key Stretching): 비밀번호와 솔트를 수천~수만 번 반복해서 해싱하여, 해커가 무차별 대입 공격(Brute Force)을 하기 어렵도록 연산 비용을 높입니다. [6] 
* 간결한 인터페이스: Node.js 내장 crypto.pbkdf2를 직접 쓰는 것보다 코드가 단순하여 다루기 쉽습니다. [4, 7, 8] 

------------------------------
## 💻 실제 사용 예시
사용자 비밀번호를 암호화하거나 검증할 때 다음과 같이 사용합니다: [2] 

const hasher = require('pbkdf2-password')();
// 1. 회원가입 시: 비밀번호 암호화
hasher({ password: "myPassword123" }, function(err, pass, salt, hash) {
    if (err) throw err;
    
    console.log(salt); // 자동 생성된 솔트 값 (DB에 저장 필수)
    console.log(hash); // 최종 암호화된 결과물 (DB에 저장 필수)
});
// 2. 로그인 시: 비밀번호 검증 (기존 salt 재사용)
hasher({ password: "userEnteredPassword", salt: savedSalt }, function(err, pass, salt, hash) {

        console.log("로그인 성공!");
    } else {
        console.log("비밀번호 불일치!");
    }
});
