

void
이 void연산자는 표현식을 평가하고 반환 값을 버립니다.

void는 주어진 표현식을 평가하고 항상 undefined를 반환하는 연산자입니다. 
주로 웹 페이지에서 링크 클릭 시 아무런 동작도 하지 않게 하거나, 
특정 값이 undefined인지 확인할 때 사용됩니다.
핵심 기능 및 활용
1. undefined 값 반환어떤 값이 오든 무조건 undefined를 뱉어냅니다. 
변수를 정의할 때 원시값 undefined를 안전하게 얻기 위해 void 0 또는 void(0) 형태로 자주 씁니다.
console.log(void 42); // 출력: undefined2. 
javascript:void(0) (하이퍼링크 방지)
<a> 태그 클릭 시 화면이 맨 위로 올라가거나 원치 않는 페이지 이동을 막기 위해 사용됩니다. 
링크의 href 속성에 javascript:void(0)를 넣으면 아무 일도 일어나지 않습니다.
3. 화살표 함수의 암시적 반환 방지화살표 함수에서 결과를 반환하지 않으려고 할 때 쓰입니다.
const foo = () => void doSomething();
위 코드는 doSomething()을 실행하고 그 결과에 상관없이 무조건 undefined를 반환하도록 만듭니다.

typeof
연산자 typeof는 주어진 객체의 유형을 결정합니다.

typeof는 자바스크립트에서 변수나 값의 데이터 타입을 확인하여 문자열로 반환해 주는 연산자입니다.
자주 반환되는 문자열 결과는 다음과 같습니다:
"string": 문자열
"number": 숫자
"boolean": 참(true)/거짓(false)
"undefined": 값이 할당되지 않은 상태
"object": 객체(주의: null도 이 타입으로 반환됨) 및 배열(Array)
"function": 함수

예시 : 

console.log(typeof 42);
// Expected output: "number"

await
비동기 함수를 일시 중지하고 다시 시작하며, 프로미스의 완료/거부를 기다립니다.

자바스크립트에서 await는 Promise가 처리(성공 또는 실패)될 때까지 기다리게 하는 키워드입니다.
이를 통해 비동기 작업을 마치 동기 작업처럼 순차적이고 직관적으로 작성할 수 있습니다.
주요 특징과 동작 방식은 다음과 같습니다.
1. 주요 기능
실행 일시 중지 및 재개: await를 만나면 해당 비동기 작업이 끝날 때까지 기다린 후(대기), 
완료되면 결과값을 반환하고 다음 코드를 실행합니다.
스레드 차단 안 함: 기다리는 동안 자바스크립트 메인 스레드는 멈추지 않고 다른 작업을 계속 수행하므로(논블로킹), 
프로그램이 멈추지 않습니다.
2. 필수 조건
await는 반드시 async 키워드가 붙은 함수 내부에서만 사용할 수 있습니다.
보통 서버에서 데이터를 받아오거나(API 호출), 파일 읽기 등 시간이 걸리는 작업 앞에 사용합니다.
3. 간단한 예제 코드javascript
async function fetchUserData() {
  // 1. 서버에 요청을 보내고 응답이 올 때까지 대기
  const response = await fetch('https://example.com');
  
  // 2. 응답 데이터를 JSON 형태로 변환할 때까지 대기
  const user = await response.json();
  
  // 3. 변환된 데이터를 바탕으로 출력
  console.log(user.name);
}
4. 에러 처리await로 기다리던 Promise가 실패(거부)할 경우 에러가 발생하므로,
이를 처리하려면 try...catch 문을 함께 사용해야 안전합니다.javascript
async function getData() {
  try {
    const data = await fetchApi();
    console.log(data);
  } catch (error) {
    console.error('에러 발생:', error);
  }
}
