var users = {
  tj: { name: 'tj' }

var users = { tj: { name: 'tj' } };는 Node.js(JavaScript)에서 간단한 인메모리(In-Memory) 사용자 데이터베이스 역할을 하는 객체를 선언한 것입니다.
데이터베이스(MySQL, MongoDB 등)를 아직 연동하지 않은 테스트 단계나 학습용 코드에서 회원 정보를 임시로 저장하고 조회하기 위해 자주 사용하는 패턴입니다.
## 🔑 구조 및 문법적 의미

* 중첩 객체(Nested Object) 구조: users라는 대형 객체 안에, 사용자 아이디인 tj를 Key(키)로 사용하고, 그 안에 해당 사용자의 상세 정보 객체({ name: 'tj' })를 Value(값)로 매핑한 구조입니다.
* 인메모리 데이터: 서버의 메모리(RAM)에 직접 저장되므로 속도가 매우 빠릅니다. 하지만 서버를 껐다 켜면 데이터가 모두 초기화(증발)된다는 특징이 있습니다.

## 🔄 실제 활용 시나리오 (예시 플로우)
앞서 질문하신 세션, 로그인 미들웨어 흐름과 연결하면 주로 사용자 조회 및 인증에 사용됩니다.
1. 로그인 확인 (조회)
사용자가 로그인 창에 아이디로 tj를 입력하면, 서버는 이 객체를 조회하여 가입된 회원인지 확인합니다.

var username = 'tj'; // 사용자가 입력한 아이디
if (users[username]) {
    console.log('회원 정보 발견:', users[username].name); // 'tj' 출력
} else {
    console.log('존재하지 않는 회원입니다.');
}

2. 회원가입 (추가)
새로운 회원(예: 아이디 gildong)이 가입하면 users 객체에 동적으로 새로운 프로퍼티를 추가합니다.

users['gildong'] = { name: '홍길동' };
// 결과 users 객체:// {//   tj: { name: 'tj' },//   gildong: { name: '홍길동' }// }

BigInt

자바스크립트의 BigInt는 기존 숫자형(Number)이 안정적으로 표현할 수 있는 
최대치인 2⁵³-1 (약 900경)보다 큰 정수를 길이의 제한 없이 정확하게 표현하고 연산할 수 있게 해주는 8번째 새로운 원시 데이터 타입입니다.
BigInt가 등장한 이유와 의미
정밀도 한계 극복: 기존 자바스크립트의 Number 타입은 64비트 부동 소수점 방식을 사용하여 9007199254740991을 넘어가면 숫자가 누락되거나 왜곡되는 정밀도 문제가 발생합니다.
무한한 자릿수: BigInt는 메모리가 허용하는 한 무한히 큰 정수를 다룰 수 있어, 
대용량 데이터 처리, 암호화, 금융 등에서 오차 없는 정확한 정수 계산이 가능합니다.
BigInt 사용 방법숫자 뒤에 n을 붙이거나 BigInt() 함수를 호출하여 생성합니다.
javascript
const bigIntNum = 9007199254740999n;
const bigIntFromFunc = BigInt("9007199254740999");

Decimal

자바스크립트에는 고유한 Decimal 기본 타입이 없으며, 대신 모든 숫자를 64비트 부동소수점(Number 타입)으로 처리합니다.
따라서 'Decimal'은 보통 10진수(십진법) 자체를 의미하거나, 
소수점 이하의 부동소수점 연산 오류를 해결하기 위해 사용하는 별도의 라이브러리를 뜻합니다.
1. 10진수 (십진법, Decimal)우리가 일상에서 사용하는 0~9로 이루어진 수 체계를 뜻합니다. 
자바스크립트는 내부적으로 이진수(Binary)를 사용하지만, 
개발자가 코드를 작성할 때는 10진수로 숫자를 표현합니다.
2. 부동소수점 오차 문제자바스크립트의 Number 타입은 10진수 소수를 2진수로 변환하여 저장합니다. 
이때 무한소수가 발생하여 근사값으로 저장되기 때문에, 
흔히 발생하는 연산 오류의 원인이 됩니다.
발생 예시: 0.1 + 0.2 의 결과값은 0.3이 아니라 0.30000000000000004가 됩니다.
3. 해결책: Decimal 라이브러리 사용이러한 금융/정밀 계산 오차를 해결하기 위해 자바스크립트에서는 외부 라이브러리의 Decimal 클래스를 도입하여 사용합니다.
숫자를 문자열 형태로 다루어 정확한 10진수 연산을 보장합니다.
관련 라이브러리: decimal.js 또는 bignumber.js사용 예시:javascript// 라이브러리 활용 예시
const x = new Decimal(0.1);
const y = new Decimal(0.2);
console.log(x.plus(y).toNumber()); // 0.3

async

자바스크립트에서 async는 'Asynchronous(비동기)'의 약자로, 
특정 함수가 비동기적으로 작동하도록 선언하는 키워드입니다.
주요 특징과 동작 방식은 다음과 같습니다:
Promise 반환: 
async 키워드가 붙은 함수는 항상 Promise 객체를 반환합니다. 
만약 함수 내부에서 반환하는 값이 Promise가 아니더라도, 
자바스크립트는 암묵적으로 값을 Promise.resolve()로 감싸서 반환합니다.
await 사용 가능: async 함수 내부에서는 await 키워드를 사용할 수 있습니다.
await는 Promise가 처리될 때까지(결과가 나올 때까지) 함수의 실행을 일시 중지시키고, 
작업이 끝나면 다시 실행을 이어가게 만듭니다.
가독성 향상: 복잡한 Promise 체이닝(.then) 대신, 
비동기 작업을 마치 위에서 아래로 실행되는 동기 코드처럼 읽기 쉽게 작성할 수 있게 해줍니다.
기본 사용 예시javascript// 
async 키워드로 비동기 함수 선언
async function fetchData() {
  // await를 사용하여 서버에서 데이터를 받아올 때까지 대기
  let response = await fetch('https://example.com');
  let data = await response.json();
  
  return data; // 암묵적으로 Promise 객체 반환
}

fetchData().then(data => console.log(data));

eval

eval()은 문자열로 된 코드를 실행하는 내장 함수입니다. 
문자열을 인자로 받아 실제 자바스크립트 명령문이나 표현식으로 평가(Evaluate)하여 결과를 반환합니다.
예를 들어, "2 + 2"라는 문자열을 넘기면 숫자 4를 계산해 줍니다.
javascript
console.log(eval("2 + 2")); // 출력: 4

let x = 10;
eval("x = x + 5");
console.log(x); // 출력: 15
코드를 사용할 때는 주의가 필요합니다.

주의사항: 사용을 권장하지 않는 이유eval()은 강력하지만 보안에 매우 취약하고 성능을 저하시키는 주요 원인이 됩니다.
보안 취약점: 인자로 전달된 문자열이 악성 코드일 경우, 
사용자 권한으로 그대로 실행되어 해킹이나 정보 유출의 위험이 있습니다. 
(보안상 외부 데이터나 사용자 입력에는 절대 사용 금지)
느린 성능: 자바스크립트 엔진이 코드를 미리 최적화할 수 없게 만들어 실행 속도가 눈에 띄게 느려집니다.

The last escape sequence, \ followed by a line terminator, is useful for splitting a string literal across multiple lines without changing its meaning.

js

Copy
const longString =
  "This is a very long string which needs \
to wrap across multiple lines because \
otherwise my code is unreadable.";
Make sure there is no space or any other character after the backslash (except for a line break), otherwise it will not work. If the next line is indented, the extra spaces will also be present in the string's value.

You can also use the + operator to append multiple strings together, like this:

js

Copy
const longString =
