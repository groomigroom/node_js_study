




new.target
생성자에서 new.target는 에 의해 호출된 생성자를 가리킵니다 new.


new.target은 함수나 생성자가 new 연산자를 사용해 호출되었는지를 확인하는 문법입니다. 
new와 함께 호출되면 자기 자신을 반환하고, 
일반 함수로 호출되면 undefined를 반환합니다. 
이를 활용해 객체 생성 시 실수를 방지하거나, 
클래스 상속에서 인스턴스 생성을 제어할 수 있습니다.
다음은 자주 쓰이는 두 가지 활용법입니다.
1. 생성자 함수 호출 실수 방지 (Scope-Safe Constructor)new 키워드를 빠뜨리고 함수를 호출하는 실수를 막을 수 있습니다.javascript
function User(name) {
  if (!new.target) { // new 없이 호출되면 undefined
    return new User(name); // new를 붙여서 다시 호출
  }
  this.name = name;
}

const user1 = new User("Alice");
const user2 = User("Bob"); // new 키워드를 안 썼지만 에러 없이 정상 작동

2. 추상 클래스 구현특정 클래스가 직접 인스턴스화되는 것을 막고, 자식 클래스를 통해서만 생성되도록 강제할 때 사용합니다.javascript
class AbstractClass {
  constructor() {
    if (new.target === AbstractClass) {
      throw new Error("추상 클래스는 직접 인스턴스화할 수 없습니다.");
    }
  }
}

class Child extends AbstractClass {}

// const abs = new AbstractClass(); // Error 발생
const child = new Child(); // 정상 작동

import.meta
컨텍스트별 메타데이터를 JavaScript 모듈에 노출하는 객체입니다.

super
이 super키워드는 부모 객체의 생성자를 호출하거나 부모 객체의 속성에 접근할 수 있도록 합니다.

import()
이 import()구문을 사용하면 모듈을 비모듈 환경일 수도 있는 곳에 비동기적으로 동적으로 로드할 수 있습니다.
